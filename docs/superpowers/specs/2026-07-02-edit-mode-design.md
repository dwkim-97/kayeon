# 대시보드 편집 모드

작성일: 2026-07-02

## 목표

카드에 상시 노출되던 관리 컨트롤(수정/삭제/상태변경)을 편집 모드 뒤로 숨겨, 평소 화면을 깔끔하게 한다.

- 상단에 **편집 토글 버튼** 하나. 누르면 대시보드 전체가 편집 모드가 된다.
- 편집 모드일 때만 각 카드에 **수정 / 삭제 / 비활성화(활성화)** 버튼이 노출된다.
- 비활성화는 현재의 스위치 토글 대신 **명시적 텍스트 버튼**으로 바꾼다. 현재 활성이면 "비활성화", 비활성이면 "활성화" 라벨로 토글된다.

## 비목표 (YAGNI)

- DB/API 변경 없음. 상태 변경은 기존 `onStatusChange`/PATCH 흐름 그대로.
- 편집 모드는 카드별이 아니라 대시보드 전역.
- 편집 모드 상태는 localStorage에 저장하지 않는다(세션성 UI, 새로고침 시 꺼짐).

## 설계

### 1. 편집 모드 상태 & 토글 (Dashboard.tsx)

- `isEditMode: boolean` state 추가(기본 false).
- 상단 툴바(뷰 토글·필터 버튼이 있는 줄)에 편집 토글 버튼 추가.
  - OFF: 흰 배경 "편집" / ON: 보라 배경 "편집 완료".
- `isEditMode`를 `ProfileCard`에 prop으로 전달.

### 2. 카드 컨트롤 조건부 노출 (ProfileCard.tsx)

`isEditMode` prop 추가.

- **조회 모드(기본, isEditMode=false)**:
  - 유지: 선택 체크박스, 별(집착매물) 버튼, 등록자 뱃지, 비활성 grayscale 표시, 사진 슬라이더/클릭→상세모달.
  - 제거/숨김: 우측 상단 상태 스위치 제거, 하단 수정/삭제 버튼 숨김.
  - 조회 모드에서는 활성/비활성 변경 불가(편집 모드에서만 가능).
- **편집 모드(isEditMode=true)**:
  - 카드 하단에 노출: 비활성화/활성화(텍스트 버튼) · 수정 · 삭제.

### 3. 상태 버튼 동작

- 클릭 시 기존 핸들러 재사용: `onStatusChange(profile.id, isBlocked ? 'active' : 'blocked')`.
- 라벨/톤: 활성 상태 → "비활성화"(주의 톤), 비활성 상태 → "활성화"(보라 강조 톤).
- 기존 우측 상단 스위치 토글 UI는 제거.
